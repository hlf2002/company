人格测试接口说明

文档中心 开发者社区
search

查询 MBTI 测试结果

此接口允许合作伙伴手动拉取特定用户的 MBTI 测试报告数据。该接口常用于结果同步、历史记录展示以及第三方数据分析场景。

01. 接口说明

security

重要：接口安全更新

为提升数据传输安全性，我们已启用新的签名机制。所有请求必须包含有效的 access_key 和基于算法生成的 sign 参数。

02. 请求地址 (Request URL)

GET 请求
https://mbti.7373.com.cn/api/v1/query?external_user_id=UNIQUE_ID&access_key=YOUR_ACCESS_KEY&sign=GENERATED_SIGN

03. 请求参数 (Parameters)

参数名称 必填 类型 说明
access_key 是 String 平台分配的唯一身份标识
external_user_id 是 String 合作伙伴系统中的用户唯一 ID
sign 是 String 根据签名算法生成的验证串

04. 签名逻辑说明

  1. 参数排序： 将所有待发送的请求参数（排除 sign 字段本身）按照参数名的 ASCII 码从小到大进行字典序排序。
  2. 拼接参数： 将排序后的参数按照 key=value 的格式拼接成字符串，各参数间用 & 连接。
  3. 追加密钥： 在拼接串的末尾追加 &secret_key=YOUR_SECRET_KEY。
  4. 生成 MD5： 对最终拼接的字符串进行 MD5 加密（32 位小写），得到的结果即为 sign。

示例公式

sign = md5("access_key=xyz&external_user_id=123&secret_key=abc...")
