验签指南

为了保障请求的安全性和数据的完整性，所有接口请求均需包含签名（sign）参数。此机制可防止请求在传输过程中被篡改，并验证调用者的合法身份。

01. 验签步骤流程

sort_by_alpha

步骤 1：参数排序

除 sign 字段外，将所有请求参数按其参数名的 ASCII 码从小到大排序。

link

步骤 2：字符串拼接

将排序后的参数按 key=value&key2=value2... 格式拼接成一个基础待签名字符串。

key

步骤 3：追加密钥

在拼接好的字符串末尾追加 &secret_key=您的密钥。此密钥请在控制台查看。

fingerprint

步骤 4：生成摘要

对最终生成的字符串进行 MD5 加密，并将结果转换为 小写，即得到最终的签名。

02. 代码示例

sign_generator.py 
import hashlib

# 1. 准备请求参数
params = {
    "access_key": "your_access_key",
    "external_user_id": "user_123",
    "callback_url": "https://example.com/callback"
}
secret_key = "your_secret_key"

# 2. 按照 ASCII 排序
sorted_keys = sorted(params.keys())

# 3. 拼接字符串
query_string = "&".join([f"{k}={params[k]}" for k in sorted_keys])

# 4. 追加密钥
sign_str = query_string + f"&secret_key={secret_key}"

# 5. MD5 加密并转小写
sign = hashlib.md5(sign_str.encode('utf-8')).hexdigest().lower()

print(f"Generated Sign: {sign}")

03. 注意事项

warning
  • 严禁公开密钥：Secret Key 是您的最高安全凭证，请勿在前端代码（如 HTML/JS）中硬编码。
  • 服务端调用：强烈建议所有的签名操作均在您的后端服务器完成。
  • 字符编码：所有待签名内容必须统一使用 UTF-8 编码。
  • 空值处理：若参数值为空，则不参与签名。
